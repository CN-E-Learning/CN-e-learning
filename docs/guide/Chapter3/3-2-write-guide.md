# 3.2.编写交接文档以创建易于交接的环境
### 1.编写交接文档的建议    
交接文档的编写应该围绕着学校上级要求和设备已知问题和常见操作的部分解决方案来写。 

在编写时至少要注明计算机是否有已知的常见问题，是否有部分程序的密码是因为可能会影响教学而需要用户知道的，例如网络密码和二级验证密码。并注明使用这些密码的时候的要求。例如网络密码不能外泄等等。 

但是注意，你要把握好交接文档中可以说明的东西，例如设备冰点的密码，可能对学校的财产安全造成一定威胁的相关内容更不应透露。如果希望在文档里注明，应该首先询问专门人士（例如信息中心的老师）的意见。若管理人员不同意，则不应该向文档内注明。若你没没有机会与相关人员沟通，你应该遵循如下原则： 

1.“凡是管理人员或老师在第一学期初没告诉你的，就不应该写入文档内部” 

2.“多一事不如少一事。” 

按照学校要求的情况下，你应该同时注明设备上文件管理的基本要求。例如所有文件统一放在D盘，每一科都需要有专用的文件夹等等的要求。 

在这些最基础的要求完成后，应该向新同学注明一下信息中心或专门管理这些设备的老师的办公室在哪等等的事情。这样会大大减少新同学交接和未来有可能的报修的难度。 

### 2.关于有关技术老师及其他工作人员的态度
至少在笔者的所在和曾经所在的学校均并不鼓励和支持这种行为，但也许在某些地区或者学校有特殊要求，因此需要学生自行写有关交接的文档。 

以笔者的学校举例，学校的老师建议在班级离开学校之前删除全部学生按照自身需求在电脑上安装的应用，除非是老师要求留下的，全部删除即可。最好的状态是恢复电脑原来的状态。
